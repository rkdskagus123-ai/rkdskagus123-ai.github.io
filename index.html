대표 작업 4개

프로젝트를 예쁘게 포장하기보다, 어떤 문제를 어떤 구조로 줄였는지 중심으로 정리했습니다.

01

Python automation · work tool

Redmine/GitLab 주간보고 자동화

문제

매주 Redmine wiki 개인 섹션을 수동 작성하고, GitLab 커밋 이력을 별도로 확인해야 했습니다.

구현

Markdown outline을 wiki 본문으로 변환하고, 상태/날짜/진행률 누락을 게시 전에 검증하는 CLI를 만들었습니다.

증거

python redmine_weekly_report.py --validate
확인 질문 없음: 상태, 진행률, 날짜가 모두 채워져 있습니다.

기술

Python, Redmine API, GitLab API, Markdown/CommonMark, PowerShell

02

AI-assisted product tool

Exam Maker / Exam AI Analyzer

문제

AI가 시험지 초안을 빠르게 만들 수는 있지만, 형식/정답 분포/중복 문항 검증이 따로 필요했습니다.

구현

Codex CLI 기반 생성 흐름, DOCX 출력, validation warning, FastAPI 분석 API와 OCR provider 경계를 구성했습니다.

증거

scripts/e2e-smoke.mjs
scripts/analyzer-pattern-summary.test.mjs
scripts/analyzer-dry-run-smoke.mjs

기술

Node.js, FastAPI, Pydantic, SQLite, Tesseract OCR, DOCX generation

04

Flutter · local-first utility

Jungdon / My Refrigerator

문제

냉장고/수납장의 물건은 위치와 유통기한이 기억에 의존하기 쉽고, 입력이 불편하면 앱 사용이 유지되지 않습니다.

구현

물품 CRUD, 위치 트리, 검색, 유통기한 목록, 바코드 스캔, 제품명 조회를 local-first 구조로 구성했습니다.
